下午两点过五分到两点半的树阴

这篇短文主要发表了对学校基建的负面意见，所以不只是植树计划的问题，不过每当我去上下午第一节课的时候，植树计划的问题确实是最大的问题

桂林的天气像个布尔值，如果东南沿海地区没有刮台风，天气就是特别毒辣的太阳，如果刮台风，天气就是以短暂的阴天为间隔的大雨，如果钻牛角尖，认为会有闪电劈到山上的雨和让人没有干袜子穿的雨是两种不同的天气，那就算三种。

假设你是个吸血鬼，从宿舍到校门口这么走一趟，就知道校园哪里该栽树了，很可惜，制定植树计划的生物里没有吸血鬼。

想到这里我气得浑身发抖，大热天的全身冷汗手脚冰凉，这个社会还能不能好了，我们吸血鬼到底要怎么活着你们才满意，眼泪不争气的流了下来，这个国到处充斥着对吸血鬼的压迫，吸血鬼何时才能真正的站起来。

艳阳天里就连走路去上课也像是某种试炼，非常令人不爽的是下课回来的时候还能看见工人们在别的地方种树。

这几天在下雨，所以我没什么要抱怨太阳的了，说点别的，图书馆的开水机，看着快坏了，三教不知道为什么总是特别热，风扇还偏偏开了也不转，等我以后发达了，一定给学校捐笔钱，专门指定用于图书馆和教学楼的基础设施维修工作

正文

有朋友问我这个昵称有什么说法，我是觉得很有意思（以至于域名都租的这个），这四个字可以有很多种解释方法，而且都很合理

也许火车的保有量上升了，或者铁路缩水了，导致火车也需要单双号限行

也许只是有个地方，那里的汽车叫火车，火车叫汽车，所以那里的（这儿叫）汽车特别快，（这儿叫）火车反而慢

也许这个昵称隐含着一个意思，表明昵称使用者不喜欢回答电车问题

也许火车能飞，因为航空管制的缘故，火车要限行

也许铁路周围的人害怕火车的噪音会惊吓到他们的老祖宗，火车只好限行了，这很有趣，你知道吗，清朝已经灭亡 108 年了

也许动车出现了，于是绿皮车会限行

也许有阿姨在洗铁路，所以火车得限行

也许大雨把铁路冲垮了，火车要限行

也许有个人像笑话里讲的那样，准备被火车撞一下，然后爬起来喊：“下一辆”

也许火车经过会导致周围区域掉帧严重，所以火车从这一区域被移除了

这个博客怎么搭建的

大致步骤

本来想着 gayhub 上的静态博客不是都好几年了么，连程序都有三四个，结果还是叫我碰到不少坑，在这里记一下，安装过程网上到处都有，我就不复读了，只列一个简单的列表

  1. 装 node

  2. 装 git

  3. 装 hexo

  4. 注册 github

  5. 添加 ssh 公钥

  6. 创建 <username>.github.io 仓库，设置一堆选项

  7. ———— 可选 ————

  8. 购买域名

  9. 设置仓库的 custom domain

  10. 添加/更改 域名的 DNS CNAME 记录

  11. ———— 可选 ————

  12. 我是如何学会停止焦虑并爱上写作的

装 Hexo

装 Hexo 的时候会有一个默认主题 Landscape 。可能是由于我的 node 不是最新的，这个默认主题不是最新版本的，不知道具体多老，反正很老，甚至不能配置 banner 。最好还是刚装完就从 这个仓库 克隆再覆盖一下

添加 ssh 公钥

我遇到了两个麻烦

  • tortoisegit 用的是 ppk 格式的密钥，它不会自动把对应的 openssh 格式的密钥放到 /.ssh/ 里面去，所以要自己转换和添加

  • 我在 gayhub 和 gaytee(coding?) 上用的是两个不同的密钥，所以还需要针对Git多账户专门做处理

设置仓库的 custom domain

这个动作其实是是替你在仓库根目录里提交了一个 CNAME 文件，hexo 只会从本地库推送，不会从远程库拉取，所以下一次 hexo d 的时候这个 CNAME 文件肯定就被覆盖了，所以得把 CNAME 复制一份放到 source/ 目录里

按照 GitHub Docs 所述，有必要先进行这一步，再配置 DNS 的 CNAME 记录

购买域名

我是从 namesilo 买的域名，如果你也想从这里买，可以用我的优惠码 train，应该能便宜 1 美元（有效期到元旦）

CNAME

CNAME 记录只能给三级子域名（就是我这样的 log.trainrestriction.top）用，如果想用二级域名（形如 trainrestriction.top）做自定义域名，就得用 A 记录，教程在GitHub Docs

https

没有 CNAME 文件时自定义域名不能用 https

大小写（不）敏感

git 默认大小写不敏感，但 GitHub page 大小写敏感，访问链接的时候可能会出问题，新建 pages 的时候一定要注意，我的那个 Bio 页面就是这种情况，文件夹名是 Bio，访问路径是 /bio，毫无疑问的 404 了

Postscript

未来我还打算给博客添加博客功能，可能是另一篇文章了，如果不很复杂就写在这里

光盘行动

经过

吃完饭看剧的时候有人过来问我能不能帮个忙，对着镜头说一些给定的提倡光盘行动的话，我拒绝了。他又说这也许会用学校的微博发出来，我又拒绝了一遍。然后他又说你看你盘子里确实把饭吃光了说明你认同这种理念那为何不给它做做宣传呢？我不知道怎么回答，所以我仓促的又拒绝了一遍，然后尽快离开了食堂。

原因

人干点事总喜欢给自己找个原因，我不是例外。
这种事情没有意义，靠一些人机械地喊口号说服不了任何人，如果想要让人意识到剩饭不好，就该请食堂的工作人员们来说，或者发些图表，说明一下这些泔水处理起来有哪些麻烦，比让学生们句读好得多
这让我想到了一些事情，以前我见过怎么喂猪，还听过一些关于把剩饭菜卖给猪养殖户的新闻
喂猪是用猪饲料兑水，然后倒进食槽里，偶尔会有从城里饭馆拉来的泔水，这里面会有一些肉，给猪喂猪肉显然太浪费了，而且我也不知道猪是不是真的会去吃。所以会先用那种夹蜂窝煤的钳子从塑料桶里把肉夹出来，留给狗吃。
剩饭的新闻好像说的是一种新的发明，申请了专利，简单来讲就是淀粉牙签，原本使用的木制牙签混到泔水里以后挑是挑不出来的，喂猪会导致猪划伤食道/胃，剩饭卖不出价。有家饭店改用淀粉做的牙签，尽管很硬，进了食道就自动溶解了，于是周围的养殖户会优先购买他们饭店的泔水。
学校里的剩饭是不是卖到喂猪的人那里去呢？如果是，为什么要担心剩饭的问题呢？如果不是，为什么不卖呢？
比起剩饭可能引发的问题，我觉得打饭外带用塑料盒可能造成的问题更大些，没有人会在用完的饭盒里塞垃圾，所以里面都是空气，但饭盒也要和其他垃圾一起放在垃圾袋里丢掉，导致往往塞两三个饭盒，垃圾袋就装不下其他垃圾了，就是说饭盒的使用促进了垃圾袋的浪费，这些东西要怎么回收？很久没有听过这种报道
外卖的塑料袋也是，太多了，让我有点不安。
快递的硬纸箱还好，不用担心回收率，这个东西回收有钱赚，保洁阿姨会专门挑出来。

10/14 更新：宿舍区的保洁阿姨会把饭盒挑出来，剩饭放在一起，盒子放在一起，不知道是有利可图还是校方命令

回收

讲着讲着讲到回收去了，干脆再补上这些：

  • 一次性电池，无线电测向那个测向机用的也是一次性电池，没有专门回收

  • 中性笔笔芯，不知道有没有人回收

  • 废旧电器，那些 ddr2 的内存，最后会怎么处理呢？断了一个针脚的 cpu ，最后会怎么处理呢？

写博客

历史记录

我以前写过博客，那时我还很小，自以为自己有很多应该拿很多稿费的东西值的发表，所以在某个平台上注册了一个账号，开始写博客。但不幸写了四五篇就没了动力，偶然想起来，又加了一篇，再偶然想起来，发现博客被锁了，遂大怒，申诉了三次，总算又解锁了，但又开始不知道写些什么。那以后除了作文，再也没写过超过 140 字的东西。
现在想来那个编辑器很难用，但是可以切换成看源代码的模式，这可能是我第一次见到 html，而且略微有点弄明白怎么给文字加粗——现在学到以后才知道这种改用 css 了，写死在 html 里显得很低级。

直接原因

你听说过「大连东软信息学院」吗？应该很出名的，镜像的速度不错，我学 Java 的时候好像就是从那里下的 Eclipse 。很有趣的事情是在空间里发说说的时候如果包含这个学院的名字，这条说说是发不出去的。很古怪，我从来没有听说过关于这个学院的坏名声。
此外，聊天也有相似的问题，而且更甚，自以为已经发出去的消息和确实已经发出去的消息，两者在视觉上完全没区别（空间里起码会发现没有访问量显示），由于这种“特性”的存在，通过qq交谈就从一种可靠的通讯协议变成了某种古怪的 udp over tcp。
空间也并不是个开放的平台，（据说）反爬很严格。在那里写的东西，传的照片，很难导出来，如果恰巧有哪一块顶上放着这些照片的硬盘挂了，也不会负责帮忙找（别的不说，起码先来个心悦三吧）
相比之下，就显得在这里写博客有很多优点了，容易组织历史文章，而且我可以不必担忧对方看不见。
有一个缺点是发出来以后网上所有人都能看见，本来写博文应该是给朋友看的，有人说这种行为有个专有名词叫受众挑选，听起来不像只有褒义的词汇，但是我觉得有些事只讲给朋友听就像告诉朋友自己的秘密一样，显得朋友很重要。静态博客没有办法设置密码，只好等有了钱用动态博客再说了。

Q:写点啥？

A:还没想好，可能没法放到社团的 wiki 上面都会放在这里，也就是平时的一些观点之类的，也有可能放一些草稿，整理好了就放到 wiki 上，把这边的删掉。
以前话说到兴头上做过一些承诺，博客第几篇要写什么这种，现在除了要写博客本身都完全忘记了，想起来再说。
