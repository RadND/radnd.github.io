js 处理二进制数据

第一层需求

之前（大概是一年以内），我在玩用 RPG Maker 所制作的游戏，没有加密，但图片素材全部都被打包成了*.rpgmvp，用压缩软件打不开，找了找发现一个可以一张一张手动解包的网页工具。有没有支持批量转换，而且自动显示的工具呢？好像有用 java 写的，网页端的没找到。那就自个儿想法写吧，定个小目标，先从读二进制文件开始。

准备工作

  • 获取文件显然要用 File API

  • 处理二进制数据需要用 ArrayBuffer

  • 二进制文件里比较简单的我就知道一个 bmp

  • 看一眼 bmp 文件的格式定义，好几个结构体，用不了 TypedArray，需要用 DataView

注：这只是一个例子，虽然不推荐，但是主流浏览器都支持在<img>标签里显示 bmp^1

读文件

然后就开始机械的读二进制数据了，教材第 32 页倒数几行对彩色版（颜色表）的讲解有问题，多了 4 个 0000

我又找到几处错误，30 页 位图信息头的定义 ，biPlanes 的注释是一句没人能看懂的「目标设备的级别，必须为 1」，实际上这个值的意思是色彩平面的数量，早些年位不够宽的时候实现更大色深用的^2，现在颜色表里那个 rgbReserverd 被拿去存 Alpha(透明度)了，32 页讲的掩码是色彩版在嵌入式平台上的的一种常见表示方式，pc 上不用^3

购买一个平板

今年桃饱网双十一活动已经开始了，我统计了一下名下的流动财产，打算买一个平板。

为什么要买

  • 带着一堆书上课不方便，而且电子书比二手书还便宜
  • 画的图，做的计算越来越复杂，纸质草稿纸不易修改，特别是平移操作，根本不可能在纸上实现
  • 比起手机，平板看视频会舒服一些，也有可能会变成电脑放视频，平板做笔记

买什么样的

我打算买一个 Windows 平板

  • 不选苹果的平板是因为我感觉它的生态太封闭，而且我现在也没有任何其余的苹果设备
  • 不选安卓的平板是因为我不想再花时间去寻找合适的看书用的 app —— 周围人已经遇到过 epub 格式在 他们的安卓平板上显示乱码的情况，而且没有解决
  • 在 Windows 上则根本不需要多想，pdf 用火狐读足矣，epub 用 calibre
  • 安卓的平板上的 qq 不能和手机同时在线

选择 Windows 而非 Android 还有一个原因，使用安卓系统的时候，我只是个卑微的普通人[^1]，Windows 上我对浏览器多少还有一点了解。以及，在谈论 Windows 平板的时候，暗含了 GNU/Linux 平板的可能，这很有趣。

具体哪个型号

一开始我想买一个便宜点的苏菲，后来我发现没有便宜点的苏菲，只有贵的和二手的两种。翻新机还好，但是最便宜的 翻新 surface go 加上触控笔要 3300，再配键盘起码也 4000 了，对比了一下流动财产的数额和产品售价，我接受了自己是个穷人的现实。搜索一番以后，我发现国内有厂家卖寨板，好像还不赖

就当花钱租教训

直到今天之前我都没有想到中国^1的跨境互联出口这么不稳定。租了一个套路云香港的服务器，丢包率最好的时候也有30%，ssh 协议还会被当作是代理服务器的混淆方法直接强行终止
好在套路云有一年限一次的 5 天之内无条件退款，我就赶紧退款走人了
希望下次运气不会这么糟，或者我该说，封锁不会这么严。

装的系统里没有自带 nano ，导致需要用 vim（不知道有没有自带 emacs ，名字太长了，没试） 来编辑 shhd.config，把 vim 当作记事本用只需要知道如下几个操作

  • 刚打开文件的时候在命令模式下，按 I 可以切换到普通模式
  • 普通模式下按 ESC 可以切换到命令模式
  • 命令模式下敲 ?${string} 可以查找 ${string}
  • 命令模式下敲 :wq 是保存并退出

下载地址:用之前要注册一个思科网络学院的账号

遇到了一些问题，路由器分的型号比较多，只有很有限的几个型号有串口(serial)，最后发现其实是因为我没有注意听讲，路由器是模块化的， 大部分型号的串口都要在配置界面的Physical框里手动安装模块获得，要想达到和实验指导书中相同的效果，需要特意安装在模块 0 插槽

路由器基本操作

几种模式的特征

特征 名称 状态转移到此模式的方法

(config) 全局模式 特权模式 configure terminal
# 特权模式 用户模式 enable
> 用户模式 特权模式 disable 理论上的初始状态，多按几下 ↵ 和 Ctrl + Z 也能到
(config-line) 线模式 全局模式 line+接口类型+接口号
(config-router) 路由模式 全局模式配置任意路由协议

找零货币问题

在美国，就像在大多数国家一样，给任何总数的零钱最好的方法是使用一个贪婪的策略——找到面额最大但少于总数的硬币，给其中一个，然后重复。例如，在美国，支付给一个人 97 美分的现金，照此策略 会得出找零 50*1 + 25*1 + 5*2 + 1*2 的方法给出的硬币总数最少

然而，也有可能出现这种贪婪策略并不总是适用的硬币系统。例如，在一个奇怪的国家里，居民们出于某种奇怪的原因，决定使用 1、12、14、63 的面值。假设你需要返还 24 美分。最好的方法是返还 2 枚 12 美分的硬币。然而，在贪心策略下，总是选择面额小于总数的最大硬币，你会选择一个 14 美分的硬币和 10 个 1 美分的硬币，总共 15 个硬币。

Serence一文弄懂递归

我有个疑问：在货币面值满足哪些条件的时候，贪心法总是能得到硬币数量最少的一种方案？充分或必要条件都行。

1、4、5、10 付 8 ， 5 1 1 1 或 4 4

我的假设是在最大（但小于要付）的面值大于要付的钱数的一半时，如果有较小面值与要付的钱数不互质，就会发生这种情况

似乎是正确的，但是怎么证明呢？而且这个假设似乎只在存在面值为 1 的硬币的情况下成立

变成子问题？ 5 付 8 到 1 付 3 。4 付 8 到 4 付 4 。这种性质具有传递性，18 会导致 10 付 18 到 1、4、5 付
