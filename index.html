开评论

在各个给静态网站提供评论功能的解决方案里面翻来翻去，有朋友建议我用 Disqus ，但是这个服务已经被墙了，我不可能只和程序员做朋友，所以选一个没有被墙的服务比较好，但用国内的服务又得来一遍 手机号实名备案 三连，干脆跳过不考虑。最后发现一个叫 gitalk 的，可以用 issue 当评论区用，就打算用这个了。

这个https://www.v2ex.com/t/535608里说这玩意可能有危险，密钥明文我是知道的，是不是真会申请这么多权限我还要看一下，其实也不要紧，公开仓库这种东西，本地肯定是有备份的

评论的认证机制比较麻烦，因为要手动开启评论区，有个办法是给经常评论的人仓库的 contributor，这样 ta 们想说话的时候能自己初始化。不过不清楚是不是会受到上一段的影响

有个 issue 标题的字符不能超过 50 的限制，光脚的不怕穿鞋的，我直接把 permalink 删成 :year/:mouth/:day 完事，反正一天不可能写这么多篇。

有可能的话我还是希望能够耦合的少一些，不是弄个<script>生成在 html 里，而是每个页面引用同一份 js 文件，每个页面往 js 里送的参数只有自己的 issue 评论链接（我这里就是 permalink），其余的数据都配置在js里（能在生成时从 theme.xxx 取更好）
没学过这东西的逻辑，不明白该怎么写，以后再说

新标签页扩展

火狐自带的新标签页很不错，但是我想看 bing.com 的壁纸，理想中的新标签页和 bing 首页差不多，但

  • 壁纸还得有
  • 底部没有这么多推送，取而代之的是我的常访问网站，也就是自带新标签页的 Top Sites
  • 顶部没有 MS 其他产品的链接
  • 右上角无所谓，放点啥我都不注意，可能放今日天气或者别的自定义部件吧
  • 右边没有那个时不时出现，挡住屏幕 1/3 ，问我要不要参与推广活动的表格
  • 右下角那个简介默认展开
  • —— 可选 ——
  • 为了通用性，最好做成搜索引擎可以自定义
  • 为了通用性，最好做成壁纸 可以自定义（但这样就不知道壁纸简介放什么了）

然后我就开始找有没有符合我这些要求的扩展了，找了两天，没找到，有朋友建议我自己写一个，不知道他是不是说着玩，不过这确实是一种方法，所以我计划有时间的时候看一看各个新标签页相关的扩展

我不打算用油猴脚本，所以不会直接 魔改（魔这个字会不会来源于modify？） 必应

New Tab Override

可以把一个 网站/HTML 文件 设置为主页，但是很可惜我没找到符合要求的网站，所以没有用

Tabliss

看起来很好看，但是和我的想法相去甚远，没有用，竖着的屏幕用这个挺不错的

这个扩展用 React + TS 写的,源代码想生成成品扩展还要再下专门的包，看的我一脸懵逼

New Tab Splash

有点我设想中的东西的雏形，壁纸从 Splash 取，有菜单栏和搜索框。热门网站没看到从哪里设置，app 和 mail 两个菜单项似乎坏掉了

New Tab Tools

有最近关闭页面和热门网站，没有搜索框，壁纸只能手动从本地选

Infinity New Tab

能满足要求，但对我来说功能太多，太重（自带了很多图标），而且非常不“清真”：

  • 不是给火狐用的，介绍里面直接说了可能会出 bug

  • 介绍剩下的部分像是直接复制粘贴的，所有的 「Chrome」 都没有换成 「Firefox」

  • 不是 FOSS ，许可证那里写 所有权利保留

下午两点过五分到两点半的树阴

这篇短文主要发表了对学校基建的负面意见，所以不只是植树计划的问题，不过每当我去上下午第一节课的时候，植树计划的问题确实是最大的问题

桂林的天气像个布尔值，如果东南沿海地区没有刮台风，天气就是特别毒辣的太阳，如果刮台风，天气就是以短暂的阴天为间隔的大雨，如果钻牛角尖，认为会有闪电劈到山上的雨和让人没有干袜子穿的雨是两种不同的天气，那就算三种。

假设你是个吸血鬼，从宿舍到校门口这么走一趟，就知道校园哪里该栽树了，很可惜，制定植树计划的生物里没有吸血鬼。

想到这里我气得浑身发抖，大热天的全身冷汗手脚冰凉，这个社会还能不能好了，我们吸血鬼到底要怎么活着你们才满意，眼泪不争气的流了下来，这个国到处充斥着对吸血鬼的压迫，吸血鬼何时才能真正的站起来。

艳阳天里就连走路去上课也像是某种试炼，非常令人不爽的是下课回来的时候还能看见工人们在别的地方种树。

这几天在下雨，所以我没什么要抱怨太阳的了，说点别的，图书馆的开水机，看着快坏了，三教不知道为什么总是特别热，风扇还偏偏开了也不转，等我以后发达了，一定给学校捐笔钱，专门指定用于图书馆和教学楼的基础设施维修工作

也许只是有个地方，那里的汽车叫火车，火车叫汽车，所以那里的（这儿叫）汽车特别快，（这儿叫）火车反而慢

也许这个昵称隐含着一个意思，表明昵称使用者不喜欢回答电车问题

也许火车能飞，因为航空管制的缘故，火车要限行

也许铁路周围的人害怕火车的噪音会惊吓到他们的老祖宗，火车只好限行了，这很有趣，你知道吗，清朝已经灭亡 108 年了

也许有阿姨在洗铁路，所以火车得限行

这个博客怎么搭建的

大致步骤

本来想着 gayhub 上的静态博客不是都好几年了么，连程序都有三四个，结果还是叫我碰到不少坑，在这里记一下，安装过程网上到处都有，我就不复读了，只列一个简单的列表

  1. 装 node

  2. 装 git

  3. 装 hexo

  4. 注册 github

  5. 添加 ssh 公钥

  6. 创建 <username>.github.io 仓库，设置一堆选项

  7. ———— 可选 ————

  8. 购买域名

  9. 设置仓库的 custom domain

  10. 添加/更改 域名的 DNS CNAME 记录

  11. 更改 css 样式

  12. ———— 可选 ————

  13. 我是如何学会停止焦虑并爱上写作的

装 Hexo

装 Hexo 的时候会有一个默认主题 Landscape 。可能是由于我的 node 不是最新的，这个默认主题不是最新版本的，不知道具体多老，反正很老，甚至不能配置 banner 。最好还是刚装完就从 这个仓库 克隆再覆盖一下

添加 ssh 公钥

我遇到了两个麻烦

  • tortoisegit 用的是 ppk 格式的密钥，它不会自动把对应的 openssh 格式的密钥放到 /.ssh/ 里面去，所以要自己转换和添加

  • 我在 gayhub 和 gaytee(coding?) 上用的是两个不同的密钥，所以还需要针对Git多账户专门做处理

设置仓库的 custom domain

这个动作其实是是替你在仓库根目录里提交了一个 CNAME 文件，hexo 只会从本地库推送，不会从远程库拉取，所以下一次 hexo d 的时候这个 CNAME 文件肯定就被覆盖了，所以得把 CNAME 复制一份放到 source/ 目录里

按照 GitHub Docs 所述，有必要先进行这一步，再配置 DNS 的 CNAME 记录

购买域名

我是从 namesilo 买的域名，如果你也想从这里买，可以用我的优惠码 train，应该能便宜 1 美元（有效期到元旦）

CNAME

CNAME 记录只能给三级子域名（就是我这样的 log.trainrestriction.top）用，如果想用二级域名（形如 trainrestriction.top）做自定义域名，就得用 A 记录，教程在GitHub Docs

https

没有 CNAME 文件时自定义域名不能用 https

大小写（不）敏感

git 默认大小写不敏感，但 GitHub page 大小写敏感，访问链接的时候可能会出问题，新建 pages 的时候一定要注意，我的那个 Bio 页面就是这种情况，文件夹名是 Bio，访问路径是 /bio，毫无疑问的 404 了

更改 css 样式

landscape 主题的 css 样式源文件是 .styl 后缀，不知道用的什么技术，反正大致能看出来样子，照着改了改

不要混用「空格」和「Tab」，样式会出大问题
