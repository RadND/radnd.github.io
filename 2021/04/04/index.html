读书笔记 - MongoDB实战(第二版)

这本书快过借阅期限了，不打算再续借，所以暂时不写作业了，把这书再读一读。

NoSQL 的本意是 Not Only SQL ，不过我看这本书的时候感受到的最大的不同确实是 No SQL ，众所周知数据库里存的东西拿到程序里来总是会变成一个一个对象，所以即便数据库本身是不是这么回事，在不会直接接触的程序看来它还是能看成一个对象的集合，集合约等于一下就是数组，平时用的这些编程语言里都能写出来对象的数组，所以得出结论 —— 不一定非得用 SQL 语句去操作集合，用别的东西也行。

个人理解

MongoDB 里面，“一张表”（和关系型数据库里同种地位的那种东西，这儿叫集合）就是一个表对象，用 Json 对象来描述一个对数据库的操作，把 Json 对象传给表对象的某个方法（find update 之类的），就会执行这个动作。当然这个代表操作的 Json 对象的结构可不是瞎写的，所以还是有一门查询语言要学，但是比单独学一门 SQL 要强 —— 可能是由于找不到专门的手册类的教材，之前光是对几张表取连接就难住了我很久，更不用说除法了。而这个语言真想学，去看 MongoDB 的官方文档就行

回到这本书上来，前几章一直在讲查询语言，逐步的介绍各种关键字，接着介绍了聚合框架（aggregation framework），可以理解为链式编程，在 JS 里用过 Jquery 或者 Promise 肯定能明白我在说什么。因为不可能把所有数据取到在跑的程序里来再做筛选和计算，所以这些东西只好在数据库里就被描述为几个操作提前做完（也许这确实是数据库该做的事情？），

然后就是占全书一半篇幅的第三部分了，名字是 精通 MongoDB ，听起来就像那种存了大量数据的大企业里的某个特殊岗位才会需要了解的东西，细看一下目录，查询优化，分布式，灾备，监控全整上了，我就不看了，至少这次用不着看。

感受

算是一本教程和工具书的结合。这本书某些地方的翻译水平像是机翻，大部分像是我翻，剩下的部分像是没翻，可能这个译者没有程序员的从业经验，术语知道的比较少。搜了一下，发现早已有人吐槽过翻译

ok，总结完了，明天还回去
